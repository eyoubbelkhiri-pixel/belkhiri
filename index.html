IA · Assistants de code Décembre 2024 Le Monde Informatique
GitHub Copilot passe en accès gratuit dans VS Code
GitHub annonce une version gratuite de Copilot intégrée par défaut à VS Code — 2 000 complétions et 50 messages par mois inclus, choix entre GPT-4o et Claude 3.5 Sonnet…
Résumé
En décembre 2024, GitHub (filiale de Microsoft) lance Copilot Free : 2 000 complétions de code et 50 messages de chat par mois, intégrés directement à Visual Studio Code. Les développeurs choisissent entre les modèles Claude 3.5 Sonnet d'Anthropic et GPT-4o d'OpenAI. Auparavant, seuls les étudiants et mainteneurs open-source bénéficiaient d'un accès gratuit.
Mon analyse
L'accès gratuit rend Copilot incontournable pour les projets scolaires et personnels. Pour un recruteur, savoir utiliser et relire le code généré par l'IA est désormais attendu d'un développeur junior. La vraie compétence n'est plus d'écrire du code, mais de vérifier, comprendre et corriger les suggestions — ce qui nécessite des bases solides en PHP et C#.

Copilot rend les développeurs 55 % plus productifs, selon le COO de GitHub
Kyle Daigle, directeur des opérations de GitHub, détaille dans une interview les gains de productivité mesurés en interne grâce à l'IA générative…

IA · Sécurité du code Octobre 2024 ANSSI
ANSSI & BSI : bonnes pratiques pour les assistants de programmation IA
Les agences de cybersécurité française et allemande publient un guide conjoint sur les risques et précautions à prendre avec les outils de génération de code IA…
Résumé
L'ANSSI (France) et le BSI (Allemagne) publient en octobre 2024 le document "AI Coding Assistants". La sécurité commence au niveau du développeur, pas de l'administrateur système.

IA · Règlement européen Août 2024 CNIL
L'AI Act européen entre en vigueur — ce que ça change pour les développeurs
La CNIL explique le nouveau règlement européen sur l'IA entré en application le 1er août 2024, ses niveaux de risque et les obligations pour les équipes de développement…
Résumé
Le règlement européen sur l'IA (AI Act) est entré en vigueur le 1er août 2024. Il classe les systèmes d'IA en 4 niveaux de risque : inacceptable, élevé, spécifique, minimal. Depuis le 2 février 2025, les pratiques à risque inacceptable sont interdites. La CNIL précise que ce règlement complète le RGPD sans le remplacer. Développeurs et intégrateurs doivent documenter et qualifier leurs systèmes d'IA.
Mon analyse
L'AI Act concerne directement tout développeur qui intègre de l'IA dans une application. Si j'ajoute un chatbot ou un module de recommandation à un projet PHP, je dois évaluer son niveau de risque selon ce règlement. Ma certification CNIL RGPD est un bon point de départ. Connaître l'AI Act est un vrai avantage à mentionner en entretien — peu de juniors sont au courant.

Authentification · Sécurité 2024 CNIL & ANSSI
Mots de passe & MFA : les recommandations communes 2024 de la CNIL et de l'ANSSI
Les deux organismes officiels français publient leurs recommandations actualisées sur la gestion des mots de passe et l'authentification multifacteur…
Résumé
La CNIL et l'ANSSI ont mis à jour leurs recommandations sur les mots de passe. Points clés : minimum 12 à 16 caractères, priorité à l'entropie plutôt qu'aux changements fréquents (ancienne bonne pratique abandonnée), et obligation de proposer l'authentification multifacteur (MFA) pour les services manipulant des données sensibles. Mon analyse
Dans mes projets PHP (Chronophone, ADA Web), l'authentification était gérée avec des sessions et des mots de passe hashés en bcrypt — ce qui reste la base. Mais la recommandation de proposer le MFA est un niveau supérieur que je n'ai pas encore implémenté. C'est un axe de progression concret pour mes futurs projets, et un sujet que les jurys d'oral apprécient d'entendre aborder.

Ce que révèle cette veille sur l'avenir du développeur web
L'IA comme outil, pas comme remplaçant
Les articles sur GitHub Copilot (Le Monde Informatique) et les recommandations ANSSI convergent : l'IA amplifie les développeurs compétents, mais ne remplace pas les bases. Un junior sans fondamentaux solides produira du code IA qu'il ne comprend pas. La valeur se déplace vers la vérification critique, l'architecture et la conformité réglementaire.
La sécurité et le RGPD, une responsabilité du développeur
Les publications CNIL et ANSSI (fiches IA, panorama cybermenaces, recommandations MFA) montrent que chaque ligne de code exposée est une responsabilité légale et technique. Ma certification CNIL et la maîtrise de PDO me positionnent bien, mais le MFA et la conformité AI Act sont des axes de progression immédiats.
Du dev local au cloud : une transition inévitable
Docker, CI/CD et souveraineté numérique dessinent le profil du développeur 2025 : quelqu'un qui livre, déploie et maintient sa propre application. Cette culture DevOps — peu présente en BTS SIO — est la principale lacune à combler rapidement pour être opérationnel dès le premier emploi.

Cette veille de 16 articles répartis sur 4 axes thématiques, s'appuyant sur des sources françaises officielles (CNIL, ANSSI) et de la presse technique de référence (Le Monde Informatique, Developpez.com, Journal du Net), confirme une tendance centrale : le développeur web de 2025 doit être un profil hybride, à la croisée du développement applicatif, de la sécurité et des pratiques DevOps.

Pour ma carrière post-BTS SIO SLAM, cette veille m'a permis d'identifier trois axes de développement prioritaires : approfondir Laravel et PHP 8.4, acquérir les fondamentaux Docker et CI/CD, et maîtriser la conformité RGPD/AI Act dans les projets intégrant de l'IA. Ces compétences correspondent précisément aux attentes des offres d'emploi junior full-stack analysées en parallèle.

Recommandations & pistes d'application concrètes
Intégrer GitHub Copilot Free dans les projets perso pour pratiquer la revue critique du code généré et développer un regard expert sur les suggestions IA.
Containeriser le projet ADA Web avec Docker Compose pour apprendre les bases du déploiement reproductible en conditions réelles.
Implémenter le MFA sur un projet sandbox PHP pour maîtriser l'authentification forte, recommandée par la CNIL et l'ANSSI.
Mettre en place un pipeline GitHub Actions avec tests PHPUnit automatisés sur les projets de portfolio existants.
Lire les 13 fiches CNIL sur l'IA et le RGPD pour être capable de documenter correctement tout traitement de données dans mes futurs projets professionnels.
Préparer la certification AWS Cloud Practitioner comme première étape de montée en compétences cloud après le BTS, compétence très demandée en 2025.
